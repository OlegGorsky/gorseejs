# DX Feedback Loop

This document defines how recurring adoption friction turns into deterministic product paths for Gorsee as a mature product.

## Feedback Loop

When the same pain point appears repeatedly:

1. identify the recurring friction
2. determine whether it belongs in docs, scaffold output, examples, or policy
3. convert it into one canonical answer
4. add tests or scripts when the answer is product-critical

## Common Inputs

- repeated scaffold mistakes
- workspace/monorepo confusion
- deploy target confusion
- compatibility import drift
- auth/cache/data setup drift

## Product Rule

Repeated friction should become product structure, not recurring tribal knowledge.
