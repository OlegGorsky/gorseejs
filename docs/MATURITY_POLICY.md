# Maturity Policy

This document defines the long-tail maturity standard for Gorsee as a mature product.

## Purpose

Long-tail maturity work exists to keep the framework disciplined as it grows.

It is not a permission slip to add vague “future-proofing” abstractions. It is a standard for keeping explicit product quality under growth pressure.

## Core Rules

- support claims must stay aligned with validated targets
- release discipline must stay aligned with product-surface changes
- examples and recipes must remain product references, not stale demos
- compatibility surfaces must not expand without deliberate migration reasoning
- dependency growth must remain minimal and justified
- benchmark and release evidence must continue to back public claims
- baseline top-tier completion is governed by `docs/TOP_TIER_EXIT_GATE.md`, not by recurring vague hardening passes

## Product Rule

Roadmap completion is not narrative. A phase is complete only when its standards are reflected in docs, tests, and release/CI enforcement.
