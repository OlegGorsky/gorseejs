# Roadmap Completion Policy

This document defines when a roadmap phase may be considered complete for Gorsee as a mature product.

## Completion Rule

A roadmap phase is complete only when:

1. the relevant docs exist
2. policy/tests enforce the surface where appropriate
3. CI/release wiring reflects the new contract where appropriate
4. README and scaffolds are updated if the product surface changed
5. the roadmap wording is updated after the deliverable is actually finished

## Not Enough

These are not enough on their own:

- a draft document with no enforcement
- a test with no product-facing docs
- a narrative claim with no code or policy support
- a roadmap checkbox updated before the work is complete

## Product Rule

Roadmap phases should reflect finished product moves, not aspirational wording.
