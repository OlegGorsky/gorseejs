# Starter Failures

This document defines the common failure paths for scaffolded Gorsee apps as a mature product.

## Most Common Starter Mistakes

- leaving `APP_ORIGIN` on a placeholder or local-only value
- enabling proxy trust without a real trusted proxy hop
- assuming route middleware automatically protects RPC
- widening cache scope before verifying the route is non-personalized
- importing from root `gorsee` instead of `gorsee/client` or `gorsee/server`

## What New Teams Should Read First

After scaffolding an app, the minimum reference set is:

- `FRAMEWORK.md`
- `docs/CANONICAL_RECIPES.md`
- `docs/RUNTIME_FAILURES.md`
- `docs/RUNTIME_TRIAGE.md`
- `docs/CACHE_INVALIDATION.md`

## Failure-Prone Surfaces

Scaffolded apps should treat these areas as failure-prone until configured explicitly:

- `security.origin`
- `security.proxy`
- `security.rpc.middlewares`
- deploy artifacts generated by `gorsee deploy`
- route cache intent for auth-sensitive pages

## Product Standard

Scaffolded apps inherit the same standard as the framework: not a pet project, not a toy, and not a temporary demo.
